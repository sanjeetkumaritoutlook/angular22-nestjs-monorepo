25 real JavaScript scenarios asked at senior-level interviews. Save this before your next round 👇

━━━ JavaScript Scenarios ━━━

1. A function runs fine in dev but throws in production after minification. What could cause this and how do you debug it?

2. A memory leak is reported in a long-running SPA. How do you identify the source?

3. You need to execute 10 API calls but limit concurrency to 3 at a time. How do you implement this?

4. A third-party script is blocking the main thread. How do you defer or isolate its impact?

5. NaN is appearing in a UI calculation. Walk through how you'd trace and fix it.

6. A recursive function causes a stack overflow on large inputs. How do you refactor it?

7. Implement Promise.all from scratch.

8. Throttle a scroll event handler to run at most once every 200ms — no libraries.

9. A button click sometimes fires twice. What are the possible causes and fixes?

10. Implement a publish/subscribe event bus in vanilla JavaScript.

11. A loop registers event listeners but all listeners share the same variable value. What's the bug and how do you fix it?

12. How do you safely parse and validate JSON from an untrusted third-party API?

13. You need to deeply compare two nested objects for equality. How do you approach it?

14. Date calculations produce wrong results for users in different timezones. How do you fix this?

15. Implement lazy evaluation using closures or generators.

16. A synchronous script tag blocks rendering for 3 seconds. How do you fix this without changing the server?

17. Build a task queue where async tasks execute strictly one at a time.

18. A candidate claims closures always cause memory leaks. Do you agree? When is it a real concern?

19. How do you detect and handle circular references when serializing an object to JSON?

20. How do you ensure a function runs exactly once even when called concurrently?

21. A long forEach loop freezes the browser tab. How do you chunk the work without a Web Worker?

22. Implement a fetch retry mechanism — up to 3 retries with a 1-second delay between each.

23. Track which properties of a large object are accessed at runtime. How do you use Proxy for this?

24. A function is called with the wrong context inside a callback. How do you fix it without changing the caller?

25. Implement a simple observable pattern — subscribe, unsubscribe, and emit — from scratch.


 These aren't textbook questions. Every single one reflects a real situation you'll face in production.

If you can confidently walk through your thought process on 15 of these — you're already ahead of most candidates in the room.
